# دليل البدء السريع - نظام المعاملات

## الخطوة 1: تشغيل المشروع

```bash
npm install
npm run dev
```

## الخطوة 2: تسجيل الدخول

استخدم أي بيانات للدخول (النظام حالياً في وضع Demo):
- اسم المستخدم: أي شيء
- كلمة المرور: أي شيء

## الخطوة 3: تجربة نظام المعاملات

### 📝 إنشاء طلب بدل مواصلات جديد

1. من القائمة الجانبية، اختر:
   ```
   الطلبات > طلب جديد
   ```

2. اختر "بدل مواصلات" من قائمة أنواع المعاملات

3. املأ النموذج:
   - **من**: مكتب الرياض
   - **إلى**: فرع جدة
   - **المسافة**: 950 كم
   - **وسيلة النقل**: سيارة خاصة
   - **المبلغ**: 500 ريال
   - **الفترة**: لمرة واحدة
   - **السبب**: اجتماع عمل في فرع جدة
   - **الأولوية**: عادي

4. (اختياري) ارفع ملفات كمرفقات

5. اضغط "إرسال الطلب"

### 📬 عرض المعاملة في الصادر

1. اذهب إلى:
   ```
   الطلبات > الصادر
   ```

2. ستجد المعاملة التي أنشأتها للتو

3. اضغط زر "عرض" لرؤية التفاصيل الكاملة

### 📥 محاكاة استلام المعاملة (الوارد)

لتجربة نظام الوارد والرد على المعاملات، يجب تغيير موقع المعاملة يدوياً من قاعدة البيانات:

**خيار 1: استخدام Supabase Dashboard**
1. افتح [Supabase Dashboard](https://supabase.com/dashboard)
2. اذهب إلى Table Editor > transactions
3. ابحث عن المعاملة التي أنشأتها
4. غيّر `current_location` من "صادر" إلى "وارد"
5. احفظ التغييرات

**خيار 2: محاكاة من التطبيق (قادم)**
سنضيف زر "نقل إلى الوارد" في النسخة القادمة

### 💬 الرد على المعاملة

بعد نقل المعاملة إلى الوارد:

1. اذهب إلى:
   ```
   الطلبات > الوارد
   ```

2. ستجد المعاملة في القائمة

3. اضغط زر **"رد"** بجانب المعاملة

4. ستظهر نافذة الرد مع 3 خيارات:

#### 🟢 خيار 1: الموافقة
```
- اختر "الموافقة"
- أضف تعليق: "تمت الموافقة على الطلب"
- اضغط "إرسال الرد"
```
**النتيجة**: تنتقل المعاملة للمرحلة التالية

#### 🟡 خيار 2: إعادة لرفع فاتورة
```
- اختر "إعادة لرفع فاتورة"
- أضف تعليق: "الرجاء إرفاق فاتورة الوقود"
- اضغط "إرسال الرد"
```
**النتيجة**: تعود المعاملة إلى الصادر مع طلب إرفاق الفاتورة

#### 🔴 خيار 3: رفض الطلب
```
- اختر "رفض الطلب"
- أضف تعليق: "المبلغ المطلوب يتجاوز الحد المسموح"
- اضغط "إرسال الرد"
```
**النتيجة**: يتم رفض المعاملة نهائياً

### 👀 عرض مسار المعاملة

في أي وقت، يمكنك:

1. اضغط زر "عرض" بجانب أي معاملة
2. ستظهر صفحة تفاصيل المعاملة
3. شاهد:
   - معلومات المعاملة الأساسية
   - **Timeline كامل** لجميع الخطوات
   - التعليقات والملاحظات
   - المرفقات
   - معلومات إضافية

## 🗄️ بنية قاعدة البيانات

### الجداول الرئيسية

#### 1. transactions
يحتوي على المعاملات:
```sql
- transaction_number: رقم المعاملة (TRN-xxxxx)
- title: عنوان المعاملة
- transaction_type: نوع المعاملة (بدل مواصلات، إجازة، إلخ)
- status: الحالة (قيد المعالجة، منتهية، مرفوضة)
- current_location: الموقع (صادر، وارد)
- priority: الأولوية (عادي، عاجل)
```

#### 2. transaction_workflow
يحتوي على مسار المعاملة:
```sql
- transaction_id: معرف المعاملة
- step_number: رقم الخطوة
- handler_name: اسم المعالج
- action: الإجراء المتخذ
- comments: التعليقات
- status: الحالة (تم، معلق، مرفوض)
```

#### 3. transaction_attachments
يحتوي على المرفقات:
```sql
- transaction_id: معرف المعاملة
- file_name: اسم الملف
- file_url: رابط الملف
```

## 🔧 استكشاف الأخطاء

### المعاملة لا تظهر في الوارد؟
تأكد من تغيير `current_location` إلى "وارد" في قاعدة البيانات

### لا يمكن إرسال الرد؟
تأكد من:
1. اختيار نوع الإجراء
2. كتابة تعليق
3. الاتصال بقاعدة البيانات Supabase

### خطأ في الاتصال بـ Supabase؟
تحقق من ملف `.env`:
```env
VITE_SUPABASE_URL=...
VITE_SUPABASE_ANON_KEY=...
```
