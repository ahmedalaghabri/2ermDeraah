# نظام إدارة المعاملات - دليل الاستخدام

## نظرة عامة
تم بناء نظام متكامل لإدارة المعاملات الوارد والصادر مع إمكانية الرد على المعاملات وتتبع مسارها.

## المميزات الجديدة

### 1. نموذج طلب بدل المواصلات
- نموذج متكامل لطلب بدل المواصلات
- يتضمن:
  - معلومات الموظف (الاسم، الرقم الوظيفي، القسم)
  - تفاصيل التنقل (من، إلى، المسافة، وسيلة النقل)
  - المبلغ المطلوب والفترة الزمنية
  - سبب الطلب والأولوية
  - إمكانية رفع مرفقات (فواتير، إثباتات)

### 2. صفحة الوارد (Inbox)
- عرض جميع المعاملات الواردة
- إحصائيات شاملة (إجمالي، معلقة، قيد المعالجة، مكتملة، مرفوضة)
- بحث وفلترة متقدمة
- **إمكانية الرد على المعاملات** مع 3 خيارات:
  - **الموافقة**: اعتماد الطلب ونقله للمرحلة التالية
  - **إعادة لرفع فاتورة**: طلب إرفاق فاتورة أو مستند إضافي
  - **رفض الطلب**: رفض المعاملة مع توضيح السبب

### 3. صفحة الصادر (Outbox)
- عرض جميع المعاملات الصادرة
- تتبع حالة المعاملات المرسلة
- عرض تفصيلي لكل معاملة

### 4. نافذة الرد على المعاملات
- واجهة سهلة الاستخدام للرد على المعاملات
- خيارات متعددة للإجراءات
- إمكانية إضافة تعليقات مفصلة
- رفع مرفقات إضافية
- رسائل تنبيهية لكل نوع من الإجراءات

### 5. صفحة تفاصيل المعاملة
- عرض كامل لمسار المعاملة
- timeline تفاعلي يوضح جميع الخطوات
- معلومات تفصيلية عن كل خطوة:
  - المسؤول والقسم
  - الإجراء المتخذ
  - التاريخ والوقت
  - المدة المستغرقة
  - الملاحظات
  - المرفقات
  - معلومات إضافية

## كيفية الاستخدام

### إنشاء طلب بدل مواصلات جديد
1. من القائمة الجانبية، اختر "الطلبات" > "طلب بدل مواصلات"
2. قم بتعبئة النموذج بالكامل:
   - تحديد تاريخ الطلب
   - إدخال تفاصيل التنقل (من، إلى، المسافة)
   - اختيار وسيلة النقل
   - تحديد المبلغ المطلوب
   - اختيار الفترة الزمنية
   - كتابة سبب الطلب
   - رفع الفواتير والمرفقات
3. اضغط "إرسال الطلب"
4. ستظهر المعاملة في صندوق الصادر

### استلام معاملة والرد عليها
1. افتح صفحة "الوارد" من القائمة الجانبية
2. ستجد قائمة بجميع المعاملات الواردة
3. اضغط على زر "رد" بجانب المعاملة المطلوبة
4. ستظهر نافذة الرد مع الخيارات التالية:

   **أ. الموافقة:**
   - اختر "الموافقة"
   - أضف تعليقك
   - يمكنك إرفاق مستندات إضافية
   - اضغط "إرسال الرد"
   - ستنتقل المعاملة للمرحلة التالية

   **ب. إعادة لرفع فاتورة:**
   - اختر "إعادة لرفع فاتورة"
   - اكتب طلبك بوضوح (مثال: "الرجاء إرفاق الفاتورة الأصلية")
   - اضغط "إرسال الرد"
   - ستعود المعاملة إلى صندوق الصادر لمقدم الطلب
   - سيظهر له التعليق في صندوق الوارد الخاص به

   **ج. رفض الطلب:**
   - اختر "رفض الطلب"
   - وضح سبب الرفض بشكل مفصل
   - اضغط "إرسال الرد"
   - ستنتهي المعاملة وتظهر كـ "مرفوضة"

### تتبع مسار المعاملة
1. اضغط على زر "عرض" بجانب أي معاملة
2. ستظهر صفحة تفاصيل المعاملة
3. يمكنك رؤية:
   - معلومات المعاملة الأساسية
   - مسار المعاملة الكامل (timeline)
   - جميع الخطوات والإجراءات المتخذة
   - التعليقات والملاحظات
   - المرفقات في كل مرحلة
4. يمكنك تنزيل ملخص المعاملة أو طباعتها

## البنية التقنية

### قاعدة البيانات
يستخدم النظام Supabase مع الجداول التالية:

#### transactions (المعاملات)
- معلومات المعاملة الأساسية
- الحالة والموقع الحالي
- الأولوية

#### transaction_workflow (مسار المعاملة)
- سجل جميع الخطوات
- المسؤول عن كل خطوة
- الإجراءات المتخذة
- التعليقات

#### transaction_attachments (المرفقات)
- ملفات المعاملة
- روابط التحميل

## سيناريو الاستخدام الكامل

### مثال: طلب بدل مواصلات مع إعادة لرفع فاتورة

1. **الموظف (أحمد محمد السالم):**
   - يفتح نموذج "طلب بدل مواصلات"
   - يملأ البيانات:
     - من: مكتب الرياض
     - إلى: فرع جدة
     - المسافة: 950 كم
     - وسيلة النقل: سيارة خاصة
     - المبلغ: 500 ريال
     - الفترة: لمرة واحدة
     - السبب: اجتماع عمل في فرع جدة
   - يضغط "إرسال الطلب"
   - تظهر المعاملة في صندوق الصادر بحالة "قيد المعالجة"

2. **المدير المباشر:**
   - يجد المعاملة في صندوق الوارد
   - يفتح المعاملة ويقرأ التفاصيل
   - يلاحظ عدم وجود فاتورة
   - يضغط زر "رد"
   - يختار "إعادة لرفع فاتورة"
   - يكتب: "الرجاء إرفاق فاتورة الوقود أو إيصالات السفر"
   - يضغط "إرسال الرد"

3. **الموظف (مرة أخرى):**
   - يجد إشعار في صندوق الوارد
   - يفتح المعاملة ويقرأ التعليق
   - يرفع الفاتورة المطلوبة
   - يضغط "رد" ويختار "تم الإطلاع"
   - تعود المعاملة للمدير المباشر

4. **المدير المباشر (مراجعة نهائية):**
   - يجد المعاملة مع الفاتورة المرفقة
   - يراجع الفاتورة
   - يضغط "رد" ويختار "الموافقة"
   - يكتب: "تمت الموافقة بعد التحقق من الفاتورة"
   - تنتقل المعاملة للمرحلة التالية (المالية)
